## Кто мы

**Velura** — онлайн-ателье премиального интерьерного текстиля (шторы, тюль, комплекты day-night).
Мы не маркетплейс и не масс-маркет. Мы — камерная студия, которая подбирает ткань под конкретное окно, свет и сценарий жизни клиента.

**Одна фраза:** «Текстиль, свет и фактура как язык пространства, где роскошь звучит тихо.»

## Тон голоса

### Принципы
1. **Тихая уверенность** — не продаём, а показываем. Никаких «Купи сейчас!», «Скидка только сегодня!». CTA звучат как приглашение: «Подобрать ткань», «Запросить образцы».
2. **Сенсорный язык** — описываем то, что можно потрогать и увидеть: «мягкая архитектурная складка», «рассеивает дневной свет», «плотная фактура для вечерней тишины».
3. **Экспертность без менторства** — объясняем через контекст, а не через поучения. «Для карниза 250 см обычно нужно около 4 панелей» — факт, а не урок.
4. **Интимность** — пишем как личный стилист, а не как корпорация. «Смотрим на свет, пропорции и фактуру, чтобы текстиль работал именно в вашем пространстве.»

### Словарь бренда
- **Да:** свет, фактура, ткань, драпировка, покой, приватность, глубина, сценарий, мягкий, тихая роскошь, архитектура, камерный
- **Нет:** дёшево, акция, распродажа, хит продаж, заказывайте прямо сейчас, лучшая цена, эксклюзивно, wow

### Паттерны копирайтинга
- Заголовки секций: kicker-стиль, 2-4 слова, капс — «КАТАЛОГ», «СЕРВИС», «ОПИСАНИЕ»
- Описания продуктов: 1-2 предложения, фокус на свете и фактуре, без восклицательных знаков
- CTA-кнопки: глагол + существительное, без капса — «Добавить в корзину», «Подобрать ткань»
- Editorial/Журнал: поэтичные заголовки — «Свет, смягченный тканью», «Фактура, которая успокаивает»

## Визуальный язык

### Философия
Дизайн Velura — это **тёплый минимализм с глубиной**. Каждый слой интерфейса имеет прозрачность, размытие и мягкую тень, создавая ощущение вуали из тюля, наложенной на тёплую стену. Ничего резкого, ничего кричащего. Интерфейс должен ощущаться как хорошо освещённый шоурум: спокойный, тактильный, с вниманием к деталям.

### Цветовая палитра (тема Atelier — основная)

| Роль | Значение | Описание |
|------|----------|----------|
| Background | `#F1EEEA` | Тёплый крем, как небелёный лён |
| Foreground | `#221C18` | Тёмный шоколад, почти чёрный |
| Accent | `#8E7A62` | Тёплый тауп / warm tan |
| Card | `rgba(255,252,248,0.92)` | Полупрозрачный белый |
| Border | `rgba(78,68,62,0.18)` | Еле видимые контуры |
| Muted foreground | `#4E443E` | Для вторичного текста |

**Правило:** цвета всегда используются с opacity. Карточки — `bg-card/80`, границы — `border-border/30..45`, текст — `text-foreground/82..92`. Это создаёт воздух и слоистость.

### Типографика

| Роль | Шрифт | Размер | Вес |
|------|-------|--------|-----|
| UI (основной) | **Manrope** | fluid `clamp()` | 400-600 |
| Display / заголовки | **Lora** | fluid `clamp(2rem, 5.2vw, 4.4rem)` | 430-460 |
| Kicker / labels | Manrope | 11-12px, uppercase | 600-700, tracking 0.08em |
| Body | Manrope | `clamp(1rem, 0.96rem + 0.18vw, 1.08rem)` | 400, leading 1.55 |

**Правило:** заголовки продуктов — Lora (serif), всё остальное — Manrope (sans). Uppercase только для kicker/label/meta (12px и меньше).

### Скругления и формы

| Контекст | Радиус |
|----------|--------|
| Кнопки | `10px` |
| Карточки | `8-14px` (Surface) |
| Gallery frame (mobile) | `20px` |
| Badge / pill | `full` (999px) |
| Inputs | `md` (6-8px) |
| Модальные оверлеи | `16px` |

### Тени и глубина

Три уровня:
1. **Subtle** — `0 8px 24px rgba(foreground/0.04)` — карточки в покое
2. **Soft** — `0 18px 42px rgba(foreground/0.12)` — приподнятые элементы, hover
3. **Floating** — `0 24px 60px rgba(background/0.34)` — модалки, drawer

**Правило:** тени всегда тёплые (на базе foreground, не чистый чёрный).

### Glass-morphism

Это ключевая визуальная техника проекта:
- `backdrop-blur-lg` + `bg-card/80` + `border-border/30` = **glass-panel** (базовый)
- `backdrop-blur-xl` + `bg-card/90` + `border-border/45` = **glass-panel-strong** (приподнятый)
- Всегда с `border` — стекло без обводки выглядит «потерянным»
- На hover: opacity фона растёт на 10-15%, тень усиливается

### Анимации

| Пресет | Параметры | Применение |
|--------|-----------|------------|
| `transitionQuick` | 240ms, ease `[0.16, 1, 0.3, 1]` | Hover, toggle, мелкие интеракции |
| `transitionStandard` | 380ms, ease `[0.16, 1, 0.3, 1]` | Появление карточек, переходы |
| `springSmooth` | stiffness 260, damping 28 | Layout-анимации |
| `springSnappy` | stiffness 300, damping 30 | Быстрые отклики (кнопки) |
| `springSharedElement` | stiffness 245, damping 27 | Shared layout transitions (gallery → lightbox) |

**Правило:** easing `[0.16, 1, 0.3, 1]` — фирменный. Быстрый старт, мягкое замедление. Никакого `linear` или стандартного `ease-in-out`. Всегда поддерживаем `prefers-reduced-motion`.

## Архитектура компонентов

### Стек
- **Next.js 14+ (App Router)** + React 18+ + TypeScript strict
- **Tailwind CSS** — утилитарные классы + CSS variables для токенов
- **Framer Motion** — вся анимация
- **CSS Modules** — только для сложных эффектов (glow, neon trace, celebrations)
- **Monorepo**: `packages/ui` (библиотека), `apps/web` (storefront), `apps/api` (FastAPI)
- **Config-driven**: `clients/demo/` содержит shop.json, theme.json, pages.json, catalog.json

### Паттерны компонентов
- **Slots**: `leftSlot`, `rightSlot` для гибкой компоновки (TopNav)
- **Tone variants**: `tone="default" | "subtle" | "elevated" | "ghost"` (Surface, Badge)
- **Composition**: `asChild` для Radix-подобной композиции (Button)
- **Motion-first**: компоненты анимированы по умолчанию, отключаемы через `prefers-reduced-motion`

### Breakpoints
| Точка | CSS | Роль |
|-------|-----|------|
| 0-767px | default | Мобильный: bottom nav, single-column, hamburger menu |
| 768-1279px | `md:` | Планшет: desktop grid, но без desktop dock |
| 1280px+ | `min-[1280px]:` | Desktop: full nav, dock (избранное/корзина), search inline |

### Критические layout-решения
- Mobile header: menu (☰) → brand (center) → search (🔍). Без избранного/корзины — они в bottom nav
- Bottom nav: 4 вкладки (Главная, Каталог, Корзина, Избранное), `md:hidden`
- PDP gallery: mobile = single image + thumbs; desktop = 2-col masonry grid
- Lightbox/viewer: единый компонент `ProductGallery.tsx`. Mobile path пропускает shared-return анимацию
- Scroll lock: `position: fixed; top: -scrollY` с восстановлением при закрытии

## Качество кода

### Ожидаемый уровень
- **TypeScript strict** — `tsc --noEmit` должен проходить без ошибок
- **Accessibility**: ARIA-labels на русском, `role="dialog"`, keyboard nav (Arrow keys в gallery), `focus-visible` ring
- **Performance**: `will-change` только где нужно, `requestAnimationFrame` для pan, `IntersectionObserver` для scroll-driven state
- **No prop drilling**: Zustand stores (`useCartStore`, `useFavoritesStore`) для глобального state
- **CSS**: никаких `!important`, opacity через Tailwind slash-синтаксис (`bg-card/80`), переменные через `var(--token)`

### Чего НЕ делать
- Не добавлять bright/saturated цвета — палитра нейтрально-тёплая
- Не использовать стандартные тени (`shadow-md`, `shadow-lg`) — только кастомные через переменные
- Не ставить `ease-in-out` или `linear` — только фирменный easing или spring
- Не создавать новые breakpoints — использовать существующие (sm/md/lg/min-[1280px])
- Не выносить логику из `ProductGallery.tsx` в отдельные файлы — это осознанный монолит
- Не дублировать действия между top header и bottom nav на mobile
- Не писать «КУПИТЬ», «СКИДКА», «ХИТ» — тон бренда тихий и уверенный
